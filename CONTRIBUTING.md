# Contributing

This project is not open for general contributions. The source code is
available for audit, learning, and use, but development is controlled
by the author.


## Code annotations

The codebase uses structured comment markers to highlight important
implementation details:

- **TODO** — planned work or improvements.
- **NOTE** — important architectural details, assumptions, or
non-obvious behavior that explain why the code works the way it does
and must be considered before making changes.
